# QA Bug Logger — Quality Audit Sheet

This document is the official quality control and evaluation framework for auditing the automated tickets generated by the AI QA Bug Logger. When adding new QA members, use this checklist to assess the bot's performance and accuracy.

---

## 📋 Ticket Quality Evaluation Matrix

For each ticket audited, score the bot across these 4 key dimensions (Scale: 1 to 5, where 5 is Perfect).

### 1. Visual Context Extraction (Video/Image Analysis)
*   [ ] **Step Fidelity**: Did the AI accurately capture the exact navigation path taken in the video?
*   [ ] **UI Interaction Capture**: Did the AI identify specific button taps, field entries, and page transitions?
*   [ ] **Actual vs. Expected**: Does the generated "Expected Behavior" accurately align with the visual bug demonstrated?
*   *Target Threshold: ≥ 4/5 Average*

### 2. Environment Recognition
*   [ ] **Device/OS Accuracy**: Did the AI correctly extract the hardware (e.g., *Samsung S23 Ultra*) and OS (e.g., *Android 16*) either from text or visual cues?
*   [ ] **Environment Auto-detection**: Was the correct environment assigned (STAGE, PROD, DEV)?
*   *Target Threshold: Perfect Match required for 95% of tickets*

### 3. Categorization & Classification
*   [ ] **Project Routing**: Was the ticket assigned to the correct OpenProject numeric identifier (`3` for Android, `85` for iOS, etc.)?
*   [ ] **Bug Type Accuracy**: Did the LLM accurately categorize the issue type (e.g., *Functional/Logical* vs *UI/UX*)?
*   [ ] **Priority Sizing**: Does the inferred Priority match the severity of the bug?
*   *Target Threshold: ≥ 90% Accuracy*

### 4. Payload Integrity & Automation
*   [ ] **Asset Retention**: Is the original screen recording/screenshot attached to the ticket files?
*   [ ] **Markdown Cleanliness**: Is the ticket free of AI conversational filler (no "Sure, here is the bug...") and formatted strictly using standard headers?
*   [ ] **Total Latency**: Did the end-to-end process finish in < 25 seconds?
*   *Target Threshold: 100% Integrity*

## 🧪 Phase-by-Phase System Health Check

Use this checklist to periodically audit the system's underlying stability as new team members use the bot.

### Phase 1: Instant Acknowledgment (Inline)
*   **Pass Condition**: Bot responds instantly (< 5 seconds) with structural extraction results.
*   **Audit Signal**:
    ```text
    2026-05-14 ... | INFO | Phase 1 INLINE: Analyzing text from [User]
    2026-05-14 ... | INFO | ✅ Phase 1 complete in [N]s
    ```

### Phase 2: Asynchronous Processing (Media)
*   **Pass Condition**: Original video is successfully downloaded and the background task runs to completion without being killed by Cloud Run CPU-throttling.
*   **Audit Signal**:
    ```text
    2026-05-14 ... | INFO | Phase 2: Enriching with [N] media items...
    2026-05-14 ... | INFO | ✅ Phase 2 complete in [N]s
    ```

### Phase 3: OpenProject Push
*   **Pass Condition**: Ticket is created AND raw media is successfully pushed via the attachment endpoint.
*   **Audit Signal**:
    ```text
    2026-05-14 ... | INFO | Creating OpenProject ticket...
    2026-05-14 ... | INFO | Uploading attachment ([X] bytes) to ticket #[Y]...
    2026-05-14 ... | INFO | ✅ Attachment uploaded successfully to ticket #[Y]
    ```

## 🚨 Critical Troubleshooting Audit Guide

If a new QA member reports a failure, follow this internal runbook:

| Symptom | Probable Cause | Immediate Audit Action |
| :--- | :--- | :--- |
| **Bot responds "Not Registered"** | User hasn't run the `/register` command or token is missing. | Ask user to run `/register <API_KEY>`. Verify in local SQLite DB. |
| **Silent crash after initial response** | Cloud Run CPU-throttled or strong task reference broke. | Navigate to `/logs` endpoint and look for Garbage Collection errors. |
| **`HTTP 422: Project can't be blank`** | An unsupported platform name caused integer mapping to fail. | Audit `OP_PROJECTS` dictionary in `config.py`. |
| **`Chat API not available`** | `service-account.json` was lost during deployment. | Check if `.gcloudignore` was modified or overridden during deployment. |
| **Processing Time > 60s** | FPS or image quality parameters increased. | Audit `gemini_client.py` and ensure frame extraction is set to `1 fps` max. |
