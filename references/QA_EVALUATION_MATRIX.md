# QA Bug Logger — Audit Evaluation Matrix

This evaluation matrix provides a weighted scoring system to audit the performance, accuracy, and robustness of the AI QA Bug Logger. It is modeled after standard software quality evaluation matrices, adapted specifically for AI-driven ticket generation.

## 1. Bug Extraction Accuracy (Weight: 30%)
*This section evaluates how accurately the AI interpreted the user's intent and visual media.*

| Evaluation Criteria | Description | Score (1-5) |
| :--- | :--- | :--- |
| **Correct navigation steps identified** | Did the AI correctly trace the user's path (business flow) through the app using the video frames? | |
| **Correct terminology** | Were IndiaMART-specific terms (e.g., BL, LMS, BMC) correctly identified and preserved? | |
| **Correct expected vs. actual behavior** | Did the bot correctly deduce the validations and what *should* have happened vs. what *did* happen? | |
| **Correct workflow sequence** | Are the steps to reproduce listed in the exact chronological order shown in the media? | |
| **No LLM hallucinations** | (No requirement misinterpretation) Did the AI refrain from inventing details or steps not present in the source media? | |

## 2. Ticket Field Coverage (Weight: 40%)
*This section evaluates the completeness of the OpenProject ticket generated by the AI.*

Calculate the percentage of correctly populated ticket fields (Title, Description, Steps, Expected/Actual, Priority, Bug Type, OS/Device Environment, and Media Attachments).

| Coverage Level | Description | Check |
| :--- | :--- | :--- |
| **95–100%** | Perfect extraction. All fields populated, media successfully attached, formatting perfectly adheres to markdown standards. | [ ] |
| **85–94%** | Minor misses (e.g., missed the specific OS version but got the platform right, or minor markdown formatting quirk). | [ ] |
| **70–84%** | Partial context capture. Core bug is understandable, but priority is misjudged or steps are slightly vague. | [ ] |
| **50–69%** | Significant gaps. Missing environment data, generic titles, or failed media attachment. Requires heavy manual editing. | [ ] |
